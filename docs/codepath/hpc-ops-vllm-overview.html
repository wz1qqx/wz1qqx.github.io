全部 codepath 页

Attention 调度与 split-kv

vLLM attention backend 的任务划分、num_splits/split-kv 决策、prefill/decode 拆分,对照 HPC-Ops 动态 Tile 调度。

MoE router/gating

fused_moe/router 的 router 工厂、topk/grouped-topk、gating 精度处理,对照 HPC-Ops 双 BF16 Router GEMM。

MoE backend 数据流

router→expert 选择→expert GEMM→合并的全链路,triton/cutlass/marlin/deep_gemm/flashinfer 各 backend 与 dispatch。

TP 通信与 RMSNorm 融合

distributed all_reduce 路径、device communicator、layernorm,对照 HPC-Ops 通信-计算一体化。

Sampler 后处理

v1/sample 解码末端采样全流程与算子串联,对照 HPC-Ops fused_sampler 2-kernel 融合。

Humming kernel 集成

外部 humming 量化包经 quantization 注册接入 MoE/Linear 的机制(早前笔记)。

Marlin kernel 集成

GPTQ/AWQ 量化的 INT4 GEMM,作用于 Linear 与 MoE expert GEMM(早前笔记)。

说明:这些页面探索的是 vLLM 现状的对应实现,HPC-Ops 是独立的算子库,文中对照旨在帮助理解 "同一推理瓶颈,vLLM 怎么做、HPC-Ops 怎么进一步优化",不代表 vLLM 已集成 HPC-Ops。
