方法论:分层搜索(代价递增)

拿到一个大模型推理优化任务,先理解请求在框架中的完整流程与组网→实现 path 的映射,再在搜索空间里按代价从低到高推进:

Tier-1 配置搜索(便宜) — 换启动 flag,路由到一条已存在且更好的 codepath/kernel,只需重启 + bench。多数"找最优启动配置"在这一层。
Tier-2 启用非默认实现 — 框架里有但默认没走的 path。
Tier-3 kernel / feature 重写(昂贵) — 仅当 Tier-1/2 榨干、且有 profiler 证据支撑时才做。

铁律:Tier-1 必须 gate Tier-3。不先确认没有现成 config 能赢,就去写新 kernel = 浪费 GPU。
